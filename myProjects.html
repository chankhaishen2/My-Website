Classification of Sunflower Blooming Stages Using CNN and Transfer Learning

Category: Artificial intelligence, Computer vision

Language: Python

Software: Google Co-Laboratory

Date: July 2025

Institution: School of Computer Sciences, Universiti Sains Malaysia

Keywords: Machine Learning, Convolutional Neural Network, Transfer Learning, Inception V3, Image Classification, TensorFlow, Dataset Preparation, Model Training, Model Fine Tuning,

This project is the coursework for Multimodal Information Retrieval subject from March 2025 to July 2025. This project involves a nearly full cycle artificial intelligence development cycle, which includes problem definition, dataset preparation, model training, model fine tuning, model comparison and model selection, but excludes model deployment. My lecturer splits the project into two parts, namely assignment and project. The assignment is the problem definition and dataset preparation, whereas the project is the remaining process of the artificial intelligence model development cycle until model selection, together with the report for all processes. In the assignment, I was grouped with Hneah Guey Ling, Loh Min Yi, Muhammed Jubarah, Liu Sixuan, Chen Ziyi, Jiang Runjiang, Zheng Chengjin and Goh Deng Vee. Our problem is about sunflower blooming stage classification into four classes (bud, partially bloom, fully bloom and wilt) based on image. In the assignment, my task was to collect 100 image of wilt sunflowers. I bought three fresh sunflowers, waited for the flowers to wilt, and take photo of the flowers. 51 of the 100 images were collected using this method, whereas the rest of the images were collected from Pinterest and Google Search. For the project, the group was halved, and I was grouped with Hneah Guey Ling, Loh Min Yi and Muhammed Jubarah. The team jointly trains, fine tune and compares a custom convolutional neural network model and transfer learning model using VGG-16, AlexNet and Inception V3 as the base model. My task was to conduct literature review and write the related work section in the report, and train and fine tune the transfer learning model with Inception V3 base model. The fine-tuning was done by experimenting different optimizers and different percentages of unfrozen convolutional layers. In the first experiment, all convolutional layers were frozen, and among Adam, Adagrad, RMS Prop and SGD, the model trained using Adam optimizer achieved the highest test and validation accuracy. In the second experiment, Adam optimizer is used, and among 0%, 10%, 15%, 20% and 25%, the overall performance of 10% unfrozen is the best (highest test validation and considerably high validation accuracy). In both experiments, the number of epochs is 10 and the learning rate is 0.0001. So, the best configuration for Inception V3 is 10% unfrozen and using Adam optimizer. I would like to thank my lecturer, Prof. Dr. Putra Sumari for his guidance.

An Electronic Commerce Website for a Book Shop using Express.js Server with Customer Churn Prediction using Neural Network

Category: Artificial intelligence, Website, Server

Languages: Javascript, HTML, CSS, Python

Software: Visual Studio Code

Date: June 2024

Institution: INTI International College Penang in collaboration with Coventry University, UK

Keywords: Node.js, Express.js, MongoDB, Jupyter Notebook, Artificial Neural Network (Feedforward), TensorFlow.js, Machine Learning, Model Training, Binary Classification

This project is my final year project done from April 2024 to June 2024. This project consists of a dummy e-commerce book shop website equipped with customer churn prediction feature powered by artificial neural network. The server is built using Express.js framework and MongoDB database. The customer churn prediction model is built using TensorFlow.js. The training data originates from https://www.kaggle.com/datasets/fridrichmrtn/user-churn-dataset. The model is pre-trained and saved on the server and used when necessary. The churn prediction model ingests 12 predictor variables for each customer. The variables are formed around customer's activity on the website, such as user session count, number of viewing item, number of adding to cart number of transactions and transaction amount. The customer churn prediction model is a simple artificial neural network with 1 input layer with 4 neurons using rectified linear unit (ReLU) activation function, 1 hidden layer with 4 neurons using ReLU activation function and 1 output layer with 1 neuron using sigmoid activation function. In this project, customer churn is defined as the customer will not visit to the website on the next month. As the result of the customer churn prediction model, the probability of churn of each customer is displayed to the administrator. The website has two types of user, namely administrator and customer. The administrator can view churn probability of each customer, add, edit, update stock and delete each item sold on the website and view all orders in the system. The customer can register themselves, view items sold (login is not required), add item to cart (login is required), place order for items which are in the cart (login is required) and view past orders (login is required). Analysing Time Complexity of Radix Sort through Experiment by Sorting Numbers and Alphabets

Category: Data structure and algorithms

Language: Python

Software: PyCharm

Date: May 2025

Institution: School of Computer Sciences, Universiti Sains Malaysia

Keywords: Time Complexity, Primitive Operation, Radix Sort, Experiment, Number Sorting, Word Sorting

This project is the coursework for the first half of Advanced Algorithms and Complexity subject from March 2025 to May 2025. The first part of the task involves implementation of radix sort for sorting positive integers with a fixed length specified by the user. The task question does not mention radix sort in words but gives an illustration of steps of radix sort and asks that the logic of the code should follow the steps in the illustration. The second part involves implementation of radix sort for sorting words with variable length by modifying the code in the first part. The third part involves finding the time complexity of radix sort in Big-O by plotting a graph of number of primitive operations vs N. The code in the first and the second parts are modified by adding primitive operation counters. For number sorting, the experiment was conducted by setting the number of digits as the fixed variable, the number of numbers (N) as the manipulated variable and the number of primitive operations as the observed variable, whereas for word sorting, the experiment was conducted by setting the maximum number of characters as the fixed variable, the number of words (N) as the manipulated variable and the number of primitive operations as the observed variable. For both experiment, it was found that the number of primitive operations increases nearly linearly when N increases. So, it can be concluded that the Big-O time complexity of the algorithm (radix sort) is O(N). I would like to thank my lecturer, Assoc. Prof. Dr. Tan Tien Ping for his guidance.

Simulation and Comparison of Bellman-Ford Algorithm and Djikstra's Algorithm for Finding Shortest Distance in Graph

Category: Data structure and algorithms

Language: Python

Software: PyCharm

Date: July 2025

Institution: School of Computer Sciences, Universiti Sains Malaysia

Keywords: Graph, Shortest Distance, Bellman-Ford Algorithm, Djikstra's Algorithm, Time Complexity, Negative Weight Handling, Negative Cycle Detection

This project is the coursework for the second half of the Advanced Algorithms and Complexity subject from May 2025 to July 2025. In this project, Bellman-Ford algorithm and Djikstra's algorithm are simulated using a graph provided in the task question and compared in terms of negative weight handling, negative cycle detection and time complexity. Both algorithms serve the same function, which is finding the shortest distance from one selected vertex to all other vertices in a graph. From the simulation, it is proven that Bellman-Ford algorithm is able to handle negative weight, whereas Djikstra's algorithm cannot. Moreover, Bellman-Ford algorithm can be modified to be able to detect and trace negative cycle, whereas Djikstra's algorithm cannot. However, by analysing the code, it is found that the time complexity of Djikstra's algorithm is O(n+m), where n is the number of vertices and m is the number of edges, which is much lower than that of Bellman-Ford algorithm (O(nm)). I would like to thank my lecturer, Assoc. Prof. Dr. Ahmad Sufril Azlan Mohamed for his guidance.

SoccerDB (Security Assessment and Enhancement on a Website)

Category: Website, Server, Web security

Languages: Javascript, HTML, CSS

Software: Visual Studio Code

Date: November 2023

Institution: INTI International College Penang in collaboration with Coventry University, UK

Keywords: Express.js, MongoDB, Third Party API, REST API, Security, Password Hashing, PBKDF2, Password Rules, Password Strength Indicator

This project is the coursework for two subjects, namely Web Application Programming Interface (API) Development (Task 1, April 2023 to July 2023) and Security (Task 2, August 2023 to November 2023). In Task 1, a website that uses a third-party API, SportsDB, as its source of data is developed. This website allows users to view the information, such as time, location and results, of any soccer event worldwide. Then, the user can save the event in bookmarks for future reference. The user can access his own bookmarks and view the list of bookmarked events, update the result of any event and remove any event from bookmark. On the other hand, the adminstrator can view all normal users and remove any normal user. In Task 2, the security aspects of the website are reviewed. The website already implements attribute-based (login is required to access user-specific bookmark) and role-based (login is required to access administrator functionality) access control. The website also already has password hashing by using Message Digest algorithm 5 (MD5). After analysing the existing website and researching the available security options, improvements are made in two aspects. Firstly, the password is hashed using Password-Based Key Derivation Function 2 (PBKDF2) before stored in the database. The purpose of password hashing is to prevent the password from being revealed in case of the database is leaked. Before being hashed, the password is also appended with a random string called salt. Adding different salt to password of different user makes rainbow table attack (pre-compute the hashed passwords) less efficient. After salting, brute force attack is still efficient due to high computing power processor. So, the password hashing is made into a computationally heavy process by applying key stretching. So, PBKDF2 is used. Secondly, password rules, namely "must have numbers", "must have upper-case and lower-cse characters", "must have special characters", and "must have at least 10 characters" are implemented and a simple password strength indicator is developed. The first three rules are implemented to overcome dictionary attack (guessing the password from dictionary words) by introducing randomness in the password. The last rule (password length) is implemented to make brute force attack harder by making the password longer. When the password is longer, the number of all possible passwords is larger. As a reminder to the user, a password strength indicator based on the compliance of each rule and the password strength is developed on the registration page. The password strength is calculated based on password length and number of possible characters. I would like to thank my lecturers, Mr. Shahriman Mohd Said (Web API Development) and Mr. R.K. Krishnamoorthy (Security) for their guidance.

This project is the coursework for Parallel and Distributed Programming subject from April 2023 to July 2023. There is one written technical report which covers the comparison between OpenMP and CUDA and the comparison between OpenMP and MPI. There are six programming tasks in the question paper. Three of the six questions are related to parallel programming whereas three other questions are related to distributed programming. For parallel programming, part A is about printing "Hello World" from different threads and specifying the number of threads through command line and source code; part B is about comparing static scheduling and dynamic scheduling in a parallel for loop executing vector addition based on the time used (the result is static scheduling is faster than dynamic scheduling because the workload of vector addition is almost equal in each iteration); part C is about comparing the time used for parallel for loop with different number of threads (and also serial) and parallelizing different parts (outetmost loop and innermost loop) in matrix multiplication (the result is outermost loop parallelization is faster than innermost loop parallelization because the number of iteration in innermost loop parallelization is thousand times more than that of outermost loop parallelization; another observation is when size of matrix is larger, parallelization is more efficient). For distributed programming, part A is about specializing the number of processes and printing "Hello" from different processes; the first part of part B is about each slave process sends a message to the master process and the master process prints the messages, whereas the second part is similar to the first part but this time each slave process sends a different message; the first part of part C is the master process sends a message to each slave process and each slave process prints the message, whereas the second part is about the explaining the usage of message tags (the master process sends a message with tag "100" but the slave process waits for message with tag "101" so the program hangs, and the task is explains why the program hangs). The report was slightly edited on March 2025 to include additional explanation on the observation of time used for matrix multiplication using different matrix sizes, number of threads and innermost loop parallelization and outermost loop parallelization.
