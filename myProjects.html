An Electronic Commerce Website for a Book Shop using Express.js Server with Customer Churn Prediction using Neural Neteork

Languages: Javascript, HTML, CSS, Python

Software: Visual Studio Code

Keywords: Node.js, Express.js, MongoDB, Jupyter Notebook, Artificial Neural Network, TensorFlow.js, Machine Learning, Binary Classification

This project is my final year project done from April 2024 to June 2024. This project consists of a dummy e-commerce book shop website equipped with customer churn prediction feature powered by artificial neural network. The website has two types of user, namely administrator and customer. The website is built using javascript (Express.js). The customer churn prediction model is built using Tensor Flow. The training data originates from https://www.kaggle.com/datasets/fridrichmrtn/user-churn-dataset. The model is pre-trained and saved on the server and used when necessary. The churn prediction is made based on customer's activity on the website, such as number of viewing item, number of adding to cart and number of transactions. SoccerDB

Languages: Javascript, HTML, CSS

Software: Visual Studio Code

Keywords: Express.js, MongoDB, Third Party API, REST API, Security, Password Hashing, PBKDF2, Password Rules, Password Strength Indicator

This project is the coursework for two subjects, namely Web Application Programming Interface (API) Development (Task 1, April 2023 to July 2023) and Security (Task 2, August 2023 to November 2023). In Task 1, a website that uses a third-party API, SportsDB, as its source of data is developed. This website allows users to view the information, such as time, location and results, of any soccer event worldwide. Then, the user can save the event in bookmarks for future reference. In Task 2, the security aspects of the website are reviewed. Improvements are made in two aspects. Firstly, the password is hashed using Password-Based Key Derivation Function 2 (PBKDF2) before stored in the database. Secondly, password rules, such as "must have numbers" or "must have special characters", are implemented and a simple password strength indicator is developed. Customer Churn Prediction

Language: Python

Software: Jupyter Notebook

Keywords: Machine Learning, Artificial Neural Network, Logistic Regression, Keras, Scikit Learn, Binary Classification, Regularization, One-Hot Encoding

This project is the coursework for Machine Learning and Related Applications subject from August 2023 to November 2023. In this project, two machine learning methods, namely logistic regression and artificial neural network, are used to predict the customers of a telecommunication company as "churner" or "non-churner". The training dataset is originated from https://www.kaggle.com/datasets/dileep070/logisticregression-telecomcustomer-churmprediction. The data is pre-processed by using one-hot-encoding to transform categorical data into numerical data. Both machine learning models are fine tuned using L1 regularization, L2 regularization and elastic net regularization. All three regularization methods are compared based on accuracy. Both machine learning models (logistic regression and artificial neural network) are compared based on accuracy, loss, precision, recall, F-score and confusion matrix. Based on the observation, in predicting customer churn, logistic regression can achieve nearly same performance as artificial neural network.
